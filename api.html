API Reference

Interactive endpoint documentation with examples and response formats

Endpoint

POST /v1/licensing

Request Body Example

{
  "ccId": {
    "resellerId": "string",
    "divisionId": "string"
  },
  "sipCredentials": {
    "username": "user123",
    "password": "secret",
    "domain": "sip.example.com",
    "proxy": "proxy.example.com",
    "register": true,
    "authUser": "authuser123",
    "outboundProxy": "outbound.example.com"
  },
  "aiMessaging": {
    "additionalMessageBundles": 5
  },
  "aiAudioServices": {
    "additionalHours": 20
  },
  "ehrEmrIntegrationSeats": 10
  // ...other license seat fields...
}

Feature Notes

  • AI Messaging: Professional and Elite seats include a base number of AI messages per agent. Use aiMessaging.additionalMessageBundles to purchase more (each bundle = 1000 messages).
  • AI Audio Services: Professional and Elite seats include a base number of AI audio hours per agent. Use aiAudioServices.additionalHours to purchase more.
  • SIP Credentials: If SIP registration is required, include the optional sipCredentials object with the following properties: username, password, domain, proxy (optional), register, authUser (optional), outboundProxy (optional).
  • EHR/EMR Integration: Use ehrEmrIntegrationSeats to specify the number of agent users with EHR/EMR integration.
